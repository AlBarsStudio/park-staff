# Руководство по боту ВКонтакте

## Назначение документа

Данный документ описывает назначение, принцип работы, список команд и правила использования бота ВКонтакте, входящего в состав системы **ParkStaff**.

Бот позволяет сотрудникам управлять своими сменами непосредственно из беседы ВКонтакте, без необходимости входа на сайт.

## 1. Назначение бота

Бот является дополнительным интерфейсом системы **ParkStaff**.

Через бота сотрудник может:

- добавлять смены на определённые даты;
- удалять ранее добавленные смены;
- просматривать свои смены за месяц или диапазон месяцев;
- проверять подключение к системе.

Бот работает с той же базой данных, что и сайт. Все данные, добавленные через бота, сразу отображаются в административной панели и в личном кабинете сотрудника.

---

## 2. Где работает бот

Бот работает в **беседе ВКонтакте**.

Особенности:

- команды отправляются в общую беседу;
- бот реагирует только на сообщения, начинающиеся с префикса `/ps`;
- остальные сообщения бот игнорирует;
- бот отвечает в ту же беседу, в которую была отправлена команда.

---

## 3. Идентификация сотрудника

При получении команды бот определяет сотрудника по **VK ID** отправителя.

### Как это работает

1. Бот получает `from_id` из входящего сообщения.
2. Проверяет наличие этого ID в маппинге `VK ID → Employee ID`.
3. Если сопоставление найдено, загружает данные сотрудника из базы.
4. Если сопоставление не найдено, бот возвращает ошибку.

### Если сотрудник не зарегистрирован

Бот ответит сообщением:

```
❌ НЕ ЗАРЕГИСТРИРОВАНЫ

🆔 VK ID: <ваш ID>

Обратитесь к администратору
```

В этом случае необходимо обратиться к администратору, чтобы ваш VK ID был добавлен в систему.

---

## 4. Общая схема работы

```text
Сотрудник пишет команду в беседу ВКонтакте
                    │
                    ▼
        Бот проверяет префикс /ps
                    │
                    ▼
    Бот определяет сотрудника по VK ID
                    │
                    ▼
      Бот разбирает команду и параметры
                    │
                    ▼
        Бот выполняет операцию в БД
                    │
                    ▼
      Бот отправляет ответ в беседу
```

## 5. Префикс команд

Все команды бота начинаются с префикса:

```
/ps
```

Сообщения без этого префикса бот полностью игнорирует.

## 6. Список команд

### 6.1. Помощь

Показывает справку по всем доступным командам.

**Формат:**

```
/ps
/ps помощь
/ps help
```

**Ответ бота:**

```
📖 КОМАНДЫ БОТА

➕ ДОБАВИТЬ:
/ps + май 5, 7, 10-15
/ps + май 7 (10-15.30)

➖ УДАЛИТЬ:
/ps - май 5, 7

📅 ПРОСМОТР:
/ps мои смены май
/ps мои смены май-август

━━━━━━━━━━━━━━━━
💡 Без времени = полная смена
```

---

### 6.2. Тест подключения

Проверяет, что бот работает и может найти вашу учётную запись.

**Формат:**

```
/ps тест
```

**Ответ, если сотрудник найден:**

```
✅ Бот работает!
👤 Иванов Иван Иванович
🆔 VK: 123456789
```

**Ответ, если сотрудник не найден:**

```
✅ Бот работает!
⚠️ Не зарегистрирован
🆔 VK: 123456789
```

---

### 6.3. Добавление смен

Добавляет одну или несколько смен на указанные даты.

**Формат:**

```
/ps + <месяц> <даты>
```

**Примеры:**

#### Добавить полные смены на отдельные даты

```
/ps + май 5, 7, 12
```

#### Добавить полные смены на диапазон дат

```
/ps + май 10-15
```

#### Комбинация отдельных дат и диапазона

```
/ps + май 5, 7, 10-15, 20
```

#### Добавить неполную смену с указанием времени

```
/ps + май 7 (10-15.30)
```

Здесь `10` — начало в 10:00, `15.30` — конец в 15:30.

#### Комбинация полных и неполных смен

```
/ps + май 5, 7 (10-15), 10-12
```

**Ответ бота при успехе:**

```
✅ Добавлено: 3

• 5 май — полная смена
• 7 май, 10:00-15:00 (5ч)
• 10 май — полная смена
```

**Ответ при ошибке:**

```
❌ Не указан месяц

Пример: /ps + май 5, 7
```

---

### 6.4. Удаление смен

Удаляет ранее добавленные смены на указанные даты.

**Формат:**

```
/ps - <месяц> <даты>
```

**Примеры:**

#### Удалить конкретные смены

```
/ps - май 5, 7
```

#### Удалить диапазон

```
/ps - май 10-15
```

**Ответ бота при успехе:**

```
✅ Удалено: 2

• 5 май — полная смена
• 7 май, 10:00-15:00 (5ч)
```

**Ответ, если смена не найдена:**

```
❌ Не найдено:
• 5 май
• 7 май
```

---

### 6.5. Просмотр смен

Показывает все ваши смены за указанный месяц или диапазон месяцев.

**Формат:**

```
/ps мои смены <месяц>
/ps мои смены <месяц>-<месяц>
```

**Примеры:**

#### Просмотр за один месяц

```
/ps мои смены май
```

#### Просмотр за диапазон месяцев

```
/ps мои смены май-август
```

**Ответ бота:**

```
📅 Ваши смены:

━━━ МАЙ ━━━
• 5 май — полная смена
• 7 май, 10:00-15:30 (5ч 30мин)
• 10 май — полная смена
• 11 май — полная смена

Всего: 4

━━━ ИЮНЬ ━━━
• 2 июнь — полная смена
• 3 июнь, 12:00-18:00 (6ч)

Всего: 2

📊 ИТОГО: 6
```

**Ответ, если смен нет:**

```
📅 Нет смен в: май
```

## 7. Формат указания дат

Бот поддерживает несколько способов указания дат.

### Отдельные даты

Перечисляются через запятую или пробел:

```
5, 7, 12
```

### Диапазон дат

Указывается через дефис:

```
10-15
```

Эквивалентно: 10, 11, 12, 13, 14, 15.

### Комбинация

```
5, 7, 10-15, 20
```

### Обязательное указание месяца

Месяц указывается текстом на русском языке.

Допустимые варианты написания:

| Полная форма | Сокращение | Родительный падеж |
|---|---|---|
| январь | янв | января |
| февраль | фев | февраля |
| март | мар | марта |
| апрель | апр | апреля |
| май | май | мая |
| июнь | июн | июня |
| июль | июл | июля |
| август | авг | августа |
| сентябрь | сен | сентября |
| октябрь | окт | октября |
| ноябрь | ноя | ноября |
| декабрь | дек | декабря |

### Определение года

- Если указанный месяц ещё не прошёл в текущем году, используется текущий год.
- Если месяц уже прошёл, дата относится к следующему году.

---

## 8. Формат указания времени

Время указывается в скобках после номера дня.

### Формат

```
<день> (<начало>-<конец>)
```

### Разделитель минут

Минуты отделяются точкой или двоеточием:

```
7 (10-15.30)
7 (10:00-15:30)
7 (10-15)
```

### Если минуты не указаны

Подставляется `:00`:

```
7 (10-15) → 10:00 - 15:00
```

### Примеры

| Ввод | Результат |
|---|---|
| `7 (10-15)` | 10:00 – 15:00 |
| `7 (10-15.30)` | 10:00 – 15:30 |
| `7 (10:00-15:30)` | 10:00 – 15:30 |
| `7 (9.30-18)` | 09:30 – 18:00 |

## 9. Полная и неполная смена

### Полная смена

Если дата указана **без времени в скобках**, бот создаёт **полную смену**:

```
/ps + май 5
```

Результат: полная смена на 5 мая.

В базе данных:

- `is_full_day = true`
- `start_time = null`
- `end_time = null`

### Неполная смена

Если после даты указано **время в скобках**, бот создаёт **неполную смену**:

```
/ps + май 5 (10-15)
```

Результат: неполная смена на 5 мая с 10:00 до 15:00.

В базе данных:

- `is_full_day = false`
- `start_time = 10:00`
- `end_time = 15:00`

### Приоритет при генерации графика

При автоматической генерации графика администратором полная смена имеет приоритет над неполной.

---

## 10. Поведение при дублировании

Если на указанную дату уже существует смена, бот **обновляет** её.

### Что это означает

- если была полная смена, а вы отправили неполную — смена заменится на неполную;
- если была неполная смена, а вы отправили полную — смена заменится на полную;
- данные всегда перезаписываются на последнюю отправленную версию.

### Техническая реализация

Используется SQL-конструкция `ON CONFLICT ... DO UPDATE`, которая:

- создаёт новую запись, если её нет;
- обновляет существующую, если на эту дату уже есть смена.

## 11. Ограничения и проверки

### Проверка регистрации

Бот не выполнит команду, если VK ID отправителя не привязан к сотруднику в системе.

### Проверка месяца

Если месяц не указан или не распознан, бот вернёт ошибку:

```
❌ Не указан месяц
```

### Проверка дат

Если даты не указаны или не распознаны, бот вернёт ошибку:

```
❌ Не указаны даты
```

### Проверка корректности даты

Бот проверяет, существует ли указанная дата (например, 31 февраля не пройдёт валидацию).

При ошибке:

```
❌ 31 февраля — некорректная дата
```

### Проверка времени

Бот проверяет, что часы и минуты находятся в допустимых диапазонах:

- часы: 0–23;
- минуты: 0–59.

При ошибке:

```
❌ 7 май — некорректное время
```

### Удаление через бота

При удалении через бота система **не применяет правило 22 часов**, которое действует на сайте.

Бот удаляет запись напрямую из базы данных по сочетанию `employee_id + work_date`.

> Администратору следует учитывать это при анализе удалённых смен.

## 12. Возможные ошибки

| Сообщение | Причина | Решение |
|---|---|---|
| `❌ НЕ ЗАРЕГИСТРИРОВАНЫ` | VK ID не привязан к сотруднику | Обратиться к администратору |
| `❌ Не указан месяц` | Месяц не распознан в тексте | Указать месяц на русском языке |
| `❌ Не указаны даты` | Даты не распознаны | Указать числа через запятую или диапазон |
| `❌ <дата> — некорректная дата` | Дата не существует | Проверить день и месяц |
| `❌ <дата> — некорректное время` | Время в неверном формате | Проверить формат часов и минут |
| `❌ Ошибка БД` | Проблема с подключением к базе | Повторить попытку позже |
| `❌ Неизвестная команда` | Команда не распознана | Ввести `/ps помощь` |
| `❌ Смены не найдены` | При удалении: смен на указанные даты нет | Проверить даты |

## 13. Связь бота с сайтом и базой данных

Бот и сайт работают с **одной и той же базой данных Supabase**.

### Таблица, с которой работает бот

Бот выполняет операции с таблицей `employee_availability`:

| Операция | SQL |
|---|---|
| Добавление | `INSERT ... ON CONFLICT DO UPDATE` |
| Удаление | `DELETE WHERE employee_id AND work_date` |
| Просмотр | `SELECT WHERE employee_id AND month` |

### Что видит администратор

Все смены, добавленные через бота, отображаются в разделе **«Управление сменами»** на сайте.

### Что видит сотрудник

Смены, добавленные через бота, видны в личном кабинете:

- в календаре;
- в списке смен;
- как основа для генерации графика.

### Двусторонняя совместимость

- смена, добавленная через сайт, видна при просмотре через бота;
- смена, добавленная через бота, видна на сайте;
- смена, удалённая через бота, исчезает с сайта;
- смена, удалённая через сайт, не будет найдена при просмотре через бота.

## 14. Рекомендации по использованию

### Для сотрудников

- используйте `/ps помощь` если забыли формат команд;
- всегда проверяйте ответ бота, чтобы убедиться что данные сохранились;
- для подробного просмотра календаря и графика используйте сайт;
- если бот не отвечает, попробуйте позже или используйте сайт;
- при ошибках регистрации обратитесь к администратору.

### Для администраторов

- учитывайте, что бот позволяет удалять смены без ограничения в 22 часа;
- при необходимости контролируйте журнал действий;
- обновляйте маппинг VK ID при добавлении новых сотрудников;
- при возникновении ошибок подключения проверяйте доступность Supabase.

### Когда лучше использовать сайт

Сайт предпочтительнее бота, если нужно:

- увидеть календарь смен в наглядном виде;
- просмотреть график от администратора;
- отметить фактическое время работы;
- посмотреть расчёт зарплаты;
- работать с приоритетами или целью изучения;
- использовать административные функции.

## 15. Техническая информация

### Платформа

- бот развёрнут как serverless-функция;
- взаимодействует с VK через Callback API;
- подключается к базе данных Supabase через Transaction Pooler.

### Используемые технологии

| Компонент | Технология |
|---|---|
| Язык | Python |
| База данных | PostgreSQL (Supabase) |
| Драйвер БД | psycopg2 |
| VK API | Callback API, версия 5.199 |
| HTTP-клиент | requests |
| Деплой | Serverless (handler-функция) |

### Подключение к базе данных

Бот подключается к базе через **Connection Pooler** Supabase, а не напрямую к PostgreSQL.

Это обеспечивает:

- стабильную работу при частых запросах;
- корректную работу в serverless-среде;
- автоматическое управление пулом соединений.

### Формат входящих событий

Бот обрабатывает два типа событий VK Callback API:

| Тип | Назначение |
|---|---|
| `confirmation` | Подтверждение сервера для VK |
| `message_new` | Новое сообщение в беседе |
