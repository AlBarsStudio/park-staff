# 🗃️ Структура базы данных

База данных развёрнута на платформе **Supabase (PostgreSQL)**.  
Аутентификация выполняется через встроенный **Supabase Auth**.

---

## Схема таблиц
auth.users (Supabase Auth)
│
├──► admins (администраторы)
└──► employees (сотрудники)
│
├──► employee_availability (заявки на смены)
├──► employee_attraction_priorities (приоритеты)
├──► employee_study_goals (цели обучения)
└──► schedule_assignments (назначения в графике)
│
└──► actual_work_log (фактическое время)

attractions (аттракционы)
activity_log (журнал действий)

---

## Таблицы

### `admins` — Администраторы

|---|---|---|

### `employee_availability` — Заявки сотрудников на смены

Сотрудник указывает, когда он **может выйти на работу**.

| Поле | Тип | Описание |
|---|---|---|
| `id` | int | Первичный ключ |
| `employee_id` | int | Ссылка на `employees.id` |
| `work_date` | date | Дата смены (YYYY-MM-DD) |
| `is_full_day` | boolean | `true` = полный день, `false` = неполный |
| `start_time` | time | Время начала (только для неполного дня) |
| `end_time` | time | Время окончания (только для неполного дня) |
| `comment` | text | Комментарий сотрудника (до 4096 символов) |
| `created_at` | timestamp | Дата создания |
| `updated_at` | timestamp | Дата последнего изменения |

**Ограничения:**
- Один сотрудник — одна запись на конкретную дату
- Нельзя добавить запись на прошедшую дату
- Удалить можно только если до даты смены > 22 часов

### `schedule_assignments` — Назначения в графике

Официальный рабочий график, составленный администратором.

| Поле | Тип | Описание |
|---|---|---|
| `id` | int | Первичный ключ |
| `work_date` | date | Дата смены |
| `employee_id` | int | Ссылка на `employees.id` |
| `attraction_id` | int | Ссылка на `attractions.id` |
| `start_time` | time | Начало смены |
| `end_time` | time | Конец смены |
| `version_type` | text | `original` или `edited` |
| `original_id` | int | Ссылка на исходное назначение (при редактировании) |
| `edited_at` | timestamp | Время последнего редактирования |
| `created_at` | timestamp | Дата создания |
| `updated_at` | timestamp | Дата последнего изменения |

### `actual_work_log` — Фактически отработанное время

Сотрудник фиксирует реальное время начала и окончания работы.

| Поле | Тип | Описание |
|---|---|---|
| `id` | int | Первичный ключ |
| `schedule_assignment_id` | int | Ссылка на `schedule_assignments.id` |
| `actual_start` | time | Фактическое время начала |
| `actual_end` | time | Фактическое время окончания |
| `created_at` | timestamp | Дата создания записи |

**Ограничения:**
- Одна запись на одно назначение (нельзя отметить дважды)
- Доступно после 22:00 в день смены или на следующий день

---

### `employee_attraction_priorities` — Приоритеты сотрудников

| Поле | Тип | Описание |
|---|---|---|
| `id` | int | Первичный ключ |
| `employee_id` | int | Ссылка на `employees.id` |
| `priority_level` | int | Уровень: `1` (высший), `2` (средний), `3` (низший) |
| `attraction_ids` | int[] | Массив ID аттракционов |
| `created_at` | timestamp | Дата создания |
| `updated_at` | timestamp | Дата последнего изменения |

**Правила:**
- Один сотрудник — максимум 3 записи (по одной на каждый уровень)
- Каждый аттракцион может быть только в одном уровне для одного сотрудника
- Наличие аттракциона в любом уровне = допуск к этому аттракциону

---

### `employee_study_goals` — Цели обучения

| Поле | Тип | Описание |
|---|---|---|
| `id` | int | Первичный ключ |
| `employee_id` | int | Ссылка на `employees.id` (уникальный) |
| `attraction_id` | int | Ссылка на `attractions.id` |
| `created_at` | timestamp | Дата создания |
| `updated_at` | timestamp | Дата последнего изменения |

**Правила:**
- Один сотрудник — одна цель обучения
- Цель выбирается из аттракционов, которых **нет** в приоритетах

---

### `activity_log` — Журнал действий

| Поле | Тип | Описание |
|---|---|---|
| `id` | int | Первичный ключ |
| `action_type` | text | Тип действия (см. ниже) |
| `description` | text | Текстовое описание действия |
| `admin_id` | int | ID администратора (если действие от админа) |

| `created_at` | timestamp | Дата и время действия |

**Типы действий (`action_type`):**

| Значение | Описание |
|---|---|
| `availability_create` | Добавление заявки на смену |
| `availability_update` | Изменение заявки на смену |
| `availability_delete` | Удаление заявки на смену |
| `schedule_create` | Создание назначения в графике |
| `schedule_update` | Изменение назначения |
| `schedule_delete` | Удаление назначения |
| `schedule_bulk_create` | Массовая генерация графика |
| `schedule_delete_by_date` | Удаление всего графика на дату |
| `attraction_create` | Добавление аттракциона |
| `attraction_update` | Изменение аттракциона |
| `attraction_delete` | Удаление аттракциона |
| `employee_update` | Изменение данных сотрудника |
| `shift_add` | Сотрудник добавил смену (через сайт) |
| `shift_delete` | Сотрудник удалил смену |
| `actual_time_log` | Сотрудник отметил фактическое время |
| `study_goal_set` | Установлена цель обучения |
| `study_goal_delete` | Удалена цель обучения |

## Аутентификация

Система использует **Supabase Auth** для управления пользователями.

### Процесс входа
Пользователь вводит email + пароль
Supabase Auth проверяет данные и возвращает JWT-сессию
Система получает auth.users.id (UUID)
Поиск в таблице admins по полю auth_uid
→ Найден → доступ как администратор (access_level 1 или 2)
Поиск в таблице employees по полю auth_uid
→ Найден → доступ как сотрудник (access_level 3)
Не найден → доступ запрещён

## Realtime подписки

Для сотрудников используются **Supabase Realtime** подписки.  
При изменении данных в БД интерфейс обновляется автоматически без перезагрузки страницы.

| Таблица | Событие |
|---|---|
| `employee_availability` | Изменение заявок на смены |
| `schedule_assignments` | Изменение графика |
| `actual_work_log` | Изменение фактического времени |
| `employee_study_goals` | Изменение цели обучения |
| `employee_attraction_priorities` | Изменение приоритетов |
