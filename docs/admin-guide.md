## 1. Вход в систему

Перейдите на сайт: **https://albarsstudio.github.io/park-staff/**

1. Введите **Email** администратора
2. Введите **Пароль**
3. Нажмите **«Войти в систему»**

После входа система автоматически определит вашу роль  
и откроет **Панель администратора**.

## 3. Управление сменами

Раздел **«Управление сменами»** — показывает все заявки сотрудников  
(когда они могут выйти на работу).

### Навигация

- Переключайте месяцы кнопками **«‹»** и **«›»**
- Выбирайте режим просмотра: **День / Неделя / Месяц**

### Статистика раздела

| Показатель | Описание |
|---|---|
| Всего смен | Количество заявок за период |
| Полный день | Заявок на полный рабочий день |
| Частичный | Заявок на неполный день |
| Сотрудников | Уникальных сотрудников с заявками |

### Поиск и фильтрация

- **Поиск** — введите имя сотрудника или дату
- **Фильтры** — фильтрация по типу: все / полный день / частичный день

### Таблица смен

Каждая строка содержит:
- **Дата** — с подсветкой выходных дней (красный индикатор)
- **Сотрудник** — имя сотрудника
- **Тип смены** — Полный день / Частичный день
- **Время** — временной диапазон
- **Действия** — кнопки управления

### Действия со сменами

#### Добавить смену
1. Нажмите кнопку **«Добавить смену»**
2. Введите имя сотрудника в поиске
3. Выберите сотрудника из выпадающего списка
4. Укажите дату
5. Выберите тип: полный день или частичный
6. При частичном — укажите время начала и конца
7. При необходимости добавьте комментарий
8. Нажмите **«Добавить»**

#### Редактировать смену
Нажмите иконку ✏️ напротив нужной смены.

> Редактирование доступно только если с даты смены прошло **не более 2 дней**.

#### Удалить смену
Нажмите иконку 🗑️ напротив нужной смены и подтвердите.

> Удаление доступно только если с даты смены прошло **не более 2 дней**.

#### Просмотр комментария
Нажмите иконку 💬 если у смены есть комментарий сотрудника.

## 4. Генератор графика

Раздел **«Генератор графика»** — автоматически составляет  
расписание работы сотрудников на выбранный день.

### Принцип работы генератора

Алгоритм учитывает:

1. **Доступность сотрудников** — заявки на смены на выбранный день
2. **Приоритеты сотрудников** — уровень 1 (высший) имеет преимущество при назначении
3. **Тип смены** — полная смена имеет приоритет над неполной
4. **Минимальный штат** — минимальное количество сотрудников для каждого аттракциона
   - В будни: `min_staff_weekday`
   - В выходные: `min_staff_weekend`
5. **Допуск** — сотрудник назначается только на аттракционы, которые есть в его приоритетах

### Как сгенерировать график

1. Выберите **дату**
2. Нажмите кнопку **«Сгенерировать»**
3. Система покажет предварительный результат
4. Просмотрите и при необходимости скорректируйте
5. Нажмите **«Сохранить график»** для публикации

> После сохранения график становится виден сотрудникам в разделе  
> **«График от администратора»**.

## 5. Ручное составление графика

Раздел **«Ручное составление»** — позволяет вручную создавать,  
редактировать и удалять назначения в графике.

### Добавить назначение вручную

1. Выберите **дату**
2. Выберите **сотрудника**
3. Выберите **аттракцион**
4. Укажите **время начала** и **время окончания**
5. Нажмите **«Добавить»**

### Редактирование назначения

- Измените аттракцион, время начала или время окончания
- Нажмите **«Сохранить»**

> При редактировании в поле `version_type` проставляется значение `edited`,  
> что позволяет отследить историю изменений.

### Удаление назначения

Нажмите кнопку 🗑️ напротив нужного назначения.

### Очистить весь день

Если нужно удалить все назначения на конкретный день — используйте  
кнопку **«Очистить день»** (если доступна в интерфейсе).

## 6. Просмотр графика смен

Раздел **«График смен»** — визуальный просмотр готового расписания.

### Режимы просмотра

| Режим | Описание |
|---|---|
| День | Один выбранный день |
| Неделя | 7 дней начиная с понедельника |
| Месяц | Все дни текущего месяца |

### Режимы отображения (только десктоп)

| Режим | Описание |
|---|---|
| 📋 Карточки | Каждый день — отдельная карточка с аттракционами и сотрудниками |
| 📊 Таблица | Классическая таблица: аттракционы по вертикали, дни по горизонтали |

На мобильном доступен только режим карточек.

### Цветовые обозначения

| Цвет | Значение |
|---|---|
| Синий | Полная смена (10:00 – 22:00) |
| Жёлтый | Частичная смена (нестандартное время) |
| Красный фон | Выходной день (суббота, воскресенье) |

### Поиск и фильтры

- **Поиск сотрудника** — найти все смены конкретного человека
- **Фильтр по аттракционам** — показать только выбранные аттракционы
- Кнопка **«Сбросить»** — убрать все фильтры

### Статистика просмотра

В шапке раздела отображается:
- Всего смен в периоде
- Количество работающих сотрудников
- Количество задействованных аттракционов

### Экспорт

> Функция экспорта графика находится в разработке.

## 7. Сотрудники

Раздел **«Сотрудники»** — список всех сотрудников парка.

### Что отображается

- ФИО сотрудника
- Возраст
- Контакты (телефон, Telegram, ВКонтакте)
- Базовая почасовая ставка
- Дата последнего входа в систему

> Добавление новых сотрудников выполняется через Supabase Auth  
> (создание пользователя) и внесение записи в таблицу `employees`.

## 8. Аттракционы

Раздел **«Аттракционы»** — управление списком аттракционов парка.

### Данные аттракциона

| Поле | Описание |
|---|---|
| Название | Наименование аттракциона |
| Мин. сотрудников (будни) | Минимально необходимое количество персонала в будние дни |
| Мин. сотрудников (выходные) | Минимально необходимое количество персонала в выходные дни |
| Коэффициент | Множитель к базовой ставке для расчёта зарплаты (например: 1.0, 1.2, 1.5) |

### Добавить аттракцион

1. Нажмите **«Добавить аттракцион»**
2. Заполните название
3. Укажите минимальный штат на будни и выходные
4. Укажите коэффициент оплаты
5. Нажмите **«Сохранить»**

### Редактировать аттракцион

Нажмите кнопку ✏️ напротив нужного аттракциона.

### Удалить аттракцион

Нажмите кнопку 🗑️ и подтвердите удаление.

> ⚠️ При удалении аттракциона все связанные назначения в графике  
> и приоритеты сотрудников могут стать некорректными.  
> Убедитесь, что аттракцион не используется в активных сменах.

## 9. Приоритеты сотрудников

Раздел **«Приоритеты»** (внутри карточки сотрудника или отдельный раздел) —  
настройка допуска сотрудников к аттракционам и порядка их назначения.

### Как работают приоритеты

- Каждый сотрудник имеет **3 уровня** приоритета
- Аттракционы в любом уровне = **есть допуск** к работе на нём
- **Уровень 1 (высший)** — сотрудника ставят в первую очередь
- **Уровень 2 (средний)** — во вторую очередь
- **Уровень 3 (низший)** — при необходимости
- Генератор графика учитывает приоритеты при автоматическом распределении

### Настройка приоритетов

1. В разделе выберите **сотрудника** из выпадающего списка
2. Для каждого уровня нажмите **«Добавить»**
3. В модальном окне отметьте нужные аттракционы
4. Нажмите **«Применить»**
5. После настройки всех уровней нажмите **«Сохранить»**

> Один аттракцион может быть только в одном уровне приоритета.  
> Аттракционы, уже добавленные в один уровень, не отображаются в других.

### Удалить аттракцион из уровня

Нажмите **×** напротив аттракциона в карточке уровня.

## Журнал активности

Все действия администраторов автоматически записываются в журнал (`activity_log`):

| Тип события | Когда записывается |
|---|---|
| `availability_create` | Добавление смены сотруднику |
| `availability_update` | Изменение смены |
| `availability_delete` | Удаление смены |
| `schedule_create` | Создание назначения в графике |
| `schedule_update` | Изменение назначения |
| `schedule_delete` | Удаление назначения |
| `schedule_bulk_create` | Массовая генерация графика |
| `attraction_create` | Добавление аттракциона |
| `attraction_update` | Изменение аттракциона |
| `attraction_delete` | Удаление аттракциона |
| `employee_update` | Изменение данных сотрудника |
