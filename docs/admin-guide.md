# Инструкция для администратора

## Назначение документа

Данный документ описывает порядок работы администратора в системе **ParkStaff**.

Система предоставляет администратору инструменты для:

- просмотра и управления сменами сотрудников;
- автоматической генерации графика;
- ручного составления графика;
- просмотра назначений по дням, неделям и месяцам;
- управления аттракционами;
- просмотра информации о сотрудниках;
- настройки приоритетов сотрудников по аттракционам.

## 1. Вход в систему

Для входа в систему перейдите по ссылке:

**Сайт:** https://albarsstudio.github.io/park-staff/

### Порядок входа

1. Введите email администратора.
2. Введите пароль.
3. Нажмите кнопку **«Войти в систему»**.

После успешного входа система определяет роль пользователя по базе данных и открывает административную панель.

### Определение роли

После авторизации система:

1. получает активную сессию через **Supabase Auth**;
2. проверяет наличие `auth_uid` в таблице `admins`;
3. если запись найдена, открывает интерфейс администратора.

## 2. Общая структура административной панели

После входа администратор попадает на главную страницу административного интерфейса.

### Верхний блок статистики

В верхней части панели отображаются ключевые показатели текущего месяца:

| Показатель | Значение |
|---|---|
| Всего сотрудников | Количество сотрудников в системе |
| Активных аттракционов | Количество аттракционов в базе |
| Смен в этом месяце | Количество записей о доступности сотрудников |
| Назначений в графике | Количество назначений в графике работы |

### Навигация по разделам

Администратору доступны следующие вкладки:

| Раздел | Назначение |
|---|---|
| Управление сменами | Работа с доступностью сотрудников |
| Генератор графика | Автоматическая генерация графика на выбранный день |
| Ручное составление | Ручное создание и редактирование графика |
| График смен | Просмотр готового графика |
| Сотрудники | Просмотр списка сотрудников |
| Аттракционы | Управление аттракционами |

### Отображение на устройствах

#### На компьютере
Вкладки отображаются горизонтально в верхней части панели.

#### На мобильном устройстве
Вкладки открываются через выпадающее или боковое меню.

## 3. Основные разделы панели администратора

Административная часть системы состоит из нескольких логически разделённых модулей:

- управление доступностью сотрудников;
- генерация расписания;
- ручная корректировка назначений;
- просмотр графика;
- управление кадровыми и справочными данными.

Каждый раздел работает с одной общей базой данных и автоматически обновляется после изменения информации.

---

## 4. Раздел «Управление сменами»

Раздел предназначен для просмотра и управления сменами сотрудников, то есть их доступностью для выхода на работу.

### Что отображается в разделе

- календарный период;
- статистика по сменам;
- таблица смен;
- поиск;
- фильтрация;
- кнопка добавления новой смены.

### Режимы просмотра

Доступны три режима:

| Режим | Назначение |
|---|---|
| День | Просмотр смен на конкретную дату |
| Неделя | Просмотр смен за семидневный период |
| Месяц | Просмотр смен за выбранный месяц |

### Статистика раздела

В блоке статистики отображаются:

| Показатель | Значение |
|---|---|
| Всего смен | Все смены за выбранный период |
| Полный день | Количество полных смен |
| Частичный | Количество неполных смен |
| Сотрудников | Количество уникальных сотрудников |

### Поиск

Поиск работает по:

- ФИО сотрудника;
- дате смены.

### Фильтрация

Можно отфильтровать смены по типу:

- все смены;
- полный день;
- частичный день.

### Таблица смен

В таблице отображаются:

| Поле | Описание |
|---|---|
| Дата | Дата смены |
| Сотрудник | ФИО сотрудника |
| Тип смены | Полный день или частичный день |
| Время | Интервал времени |
| Действия | Комментарий, редактирование, удаление |

### Особенности отображения

- выходные дни могут визуально выделяться;
- наличие комментария отображается отдельной кнопкой;
- действия по редактированию и удалению могут быть ограничены правилами системы.

## 5. Добавление смены сотруднику

Администратор может вручную добавить смену сотруднику.

### Порядок действий

1. Откройте раздел **«Управление сменами»**.
2. Нажмите кнопку **«Добавить смену»**.
3. Введите ФИО сотрудника в поле поиска.
4. Выберите сотрудника из выпадающего списка.
5. Укажите дату.
6. Выберите тип смены:
   - полный день;
   - неполный день.
7. Если выбран неполный день — укажите время начала и окончания.
8. При необходимости добавьте комментарий.
9. Нажмите кнопку **«Добавить»**.

### Проверки при сохранении

Система проверяет:

- выбран ли сотрудник;
- указана ли дата;
- указаны ли начало и окончание времени для неполной смены;
- нет ли уже смены у этого сотрудника на эту дату.

Если запись уже существует, система не позволит добавить дубликат.

---

## 6. Редактирование и удаление смен

Администратор может изменить или удалить смену через таблицу.

### Редактирование смены

Для редактирования:

1. найдите нужную строку в таблице;
2. нажмите кнопку редактирования;
3. внесите изменения;
4. сохраните запись.

### Какие данные можно изменить

- тип смены;
- время начала;
- время окончания;
- комментарий.

При редактировании дата и сотрудник не изменяются.

### Удаление смены

Для удаления:

1. нажмите кнопку удаления;
2. подтвердите действие в диалоговом окне.

### Ограничение на редактирование и удаление

Система запрещает редактирование и удаление смен, если с даты смены прошло более **2 дней**.

Это правило действует для сохранения целостности исторических данных.

## 7. Раздел «Генератор графика»

Раздел **«Генератор графика»** используется для автоматического формирования графика на определённый день.

### Назначение раздела

Этот модуль позволяет автоматически распределить сотрудников по аттракционам с учётом:

- доступности сотрудников;
- типа смены;
- допуска к аттракциону;
- уровня приоритета;
- минимальной потребности аттракциона в сотрудниках.

### Что учитывает система при генерации

#### 1. Доступность сотрудника
В генерацию попадают только сотрудники, которые указали смену на выбранную дату.

#### 2. Тип смены
Полная смена имеет приоритет над неполной.

#### 3. Допуск к аттракциону
Если аттракцион отсутствует в приоритетах сотрудника, сотрудник не считается допущенным к нему.

#### 4. Уровень приоритета
При прочих равных выше будут сотрудники, у которых аттракцион находится на более высоком уровне приоритета.

#### 5. Норматив по аттракциону
Для каждого аттракциона могут быть заданы разные значения минимального количества сотрудников:

- `min_staff_weekday` — для будней;
- `min_staff_weekend` — для выходных.

### Результат генерации

После генерации формируются назначения в таблице графика, которые затем становятся доступны:

- администратору — в разделе просмотра графика;
- сотруднику — в личном кабинете.

> Подробная логика конкретного алгоритма зависит от реализации `ScheduleGenerator` и может дорабатываться.

## 8. Раздел «Ручное составление»

Раздел **«Ручное составление»** предназначен для ручной работы с графиком, если автоматическая генерация требует корректировок.

### Варианты использования

- создание назначений вручную;
- исправление результата генерации;
- точечная настройка графика на отдельный день;
- добавление нестандартных назначений.

### Возможные действия

- выбрать сотрудника;
- выбрать аттракцион;
- указать дату;
- задать время начала и окончания;
- сохранить назначение.

### Когда использовать ручное составление

Рекомендуется использовать этот раздел, если:

- требуется исключение из общих правил;
- необходимо скорректировать автоматически созданный график;
- часть назначений нужно добавить вручную;
- нужно быстро внести изменение в уже готовый день.

## 9. Раздел «График смен»

Раздел предназначен для просмотра уже сформированного графика работы.

### Режимы просмотра

Поддерживаются три режима:

| Режим | Назначение |
|---|---|
| День | График на одну дату |
| Неделя | График за 7 дней |
| Месяц | График на месяц |

### Форматы отображения

#### Карточки
Используются на мобильных устройствах и при карточном режиме на компьютере.

Показывают:

- дату;
- аттракционы;
- назначенных сотрудников;
- частичные смены;
- количество смен на день.

#### Таблица
Используется в основном на компьютере.

Структура:

- строки — аттракционы;
- столбцы — даты;
- ячейки — сотрудники, назначенные на конкретный аттракцион в конкретный день.

### Поиск и фильтрация

В разделе доступны:

- поиск по сотруднику;
- фильтр по аттракционам;
- переключение режимов отображения;
- сброс активных фильтров.

### Отображение частичных смен

Если назначение не соответствует полному дню, система выделяет его как частичную смену и показывает время отдельно.

### Экспорт

В интерфейсе предусмотрена кнопка экспорта, однако на текущем этапе функция находится в разработке.

## 10. Раздел «Сотрудники»

Раздел **«Сотрудники»** предназначен для просмотра списка работников парка.

### Какие данные могут отображаться

- ФИО;
- возраст;
- телефон;
- Telegram;
- VK;
- Max;
- базовая почасовая ставка;
- дата последнего входа;
- дата создания записи.

### Назначение раздела

Раздел позволяет администратору:

- просматривать общую информацию по сотрудникам;
- уточнять контактные данные;
- использовать данные сотрудников при составлении графика.

---

## 11. Раздел «Аттракционы»

Раздел **«Аттракционы»** предназначен для работы со справочником аттракционов.

### Какие данные содержит аттракцион

| Поле | Описание |
|---|---|
| Название | Наименование аттракциона |
| Минимум в будни | Минимально необходимое число сотрудников в будний день |
| Минимум в выходные | Минимально необходимое число сотрудников в выходной день |
| Коэффициент | Множитель, который используется при расчёте зарплаты |

### Возможные действия

- создание аттракциона;
- редактирование аттракциона;
- удаление аттракциона.

### Зачем нужен коэффициент

Коэффициент участвует в расчёте заработной платы.

Пример:
- коэффициент `1.0` — базовый уровень;
- коэффициент `1.2` — повышенная оплата;
- коэффициент `1.5` — более высокий уровень оплаты.

## 12. Приоритеты сотрудников

Приоритеты определяют, на какие аттракционы сотрудник имеет допуск и какие аттракционы для него предпочтительнее.

### Основная логика

Если аттракцион указан у сотрудника в приоритетах, это означает:

- сотрудник имеет доступ к этому аттракциону;
- система может рассматривать его для назначения на этот аттракцион.

### Уровни приоритета

| Уровень | Значение |
|---|---|
| 1 | Высший |
| 2 | Средний |
| 3 | Низший |

### Как используются приоритеты

При автоматической генерации графика сотрудник будет предпочтительно назначаться на аттракционы с более высоким уровнем приоритета.

### Настройка приоритетов

Для настройки приоритетов:

1. выберите сотрудника;
2. откройте нужный уровень;
3. выберите доступные аттракционы;
4. сохраните изменения.

### Важное правило

Один и тот же аттракцион не должен одновременно находиться в нескольких уровнях приоритета одного сотрудника.

## 13. Логика данных и журнал действий

Система ведёт журнал административных действий.

### Что логируется

При работе администратора могут записываться события, связанные с:

- добавлением смен;
- изменением смен;
- удалением смен;
- созданием назначений;
- массовой генерацией графика;
- удалением графика по дате;
- добавлением аттракционов;
- изменением аттракционов;
- удалением аттракционов;
- изменением данных сотрудника.

### Зачем нужен журнал

Журнал действий позволяет:

- отслеживать изменения;
- понимать, какие действия были выполнены;
- анализировать административную работу;
- при необходимости восстанавливать контекст изменений.

## 14. Ограничения и правила

### Ограничения в управлении сменами

- нельзя создать дублирующую смену одному сотруднику на одну дату;
- при неполной смене обязательно должно быть указано время;
- редактирование и удаление ограничены по сроку — не более 2 дней после даты смены.

### Ограничения по данным сотрудников

Система отображает сотрудников так, как они записаны в базе данных.  
Если информация неполная или отсутствует, её необходимо корректно заполнить на уровне данных.

### Ограничения генерации графика

Корректность результата генерации зависит от качества исходных данных:

- заполненных смен сотрудников;
- актуальных приоритетов;
- корректно настроенных аттракционов;
- наличия минимальных нормативов по персоналу.

---

## 15. Рекомендации по работе

Для стабильной и удобной работы рекомендуется соблюдать следующий порядок:

### Перед генерацией графика

1. проверить, что сотрудники внесли смены на нужную дату;
2. убедиться, что аттракционы заполнены корректно;
3. проверить приоритеты сотрудников;
4. убедиться, что коэффициенты и минимальные нормативы актуальны.

### После генерации графика

1. просмотреть результат в разделе **«График смен»**;
2. проверить, все ли аттракционы закрыты по минимальному количеству сотрудников;
3. при необходимости внести корректировки вручную;
4. убедиться, что сотрудники смогут увидеть актуальные назначения в своих кабинетах.

### При работе со сменами сотрудников

- использовать комментарии для фиксации особенностей;
- не удалять исторические данные без необходимости;
- соблюдать аккуратность при массовых изменениях.
