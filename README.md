# 🎡 ParkStaff

**ParkStaff** — система управления персоналом парка аттракционов.  
Позволяет сотрудникам указывать доступность для работы, а администраторам — генерировать и управлять графиком смен.

## 📋 О проекте

Система состоит из трёх взаимосвязанных компонентов:
┌─────────────────────────────────────────────────────┐
│ База данных │
│ (Supabase) │
└──────────────┬──────────────────┬───────────────────┘
│ │
▼ ▼
┌───────────────────┐ ┌──────────────────┐
│ Сайт │ │ Бот ВКонтакте │
│ (ParkStaff Web) │ │ (в беседе) │
└───────────────────┘ └──────────────────┘

text


- **База данных (Supabase)** — централизованное хранилище всех данных
- **Сайт** — основной интерфейс для сотрудников и администраторов
- **Бот ВКонтакте** — альтернативный способ добавления смен через команды в беседе

## ✨ Возможности

### Для сотрудников
- 📅 Добавление и удаление доступных смен через календарь
- 🕐 Выбор полной или неполной смены с указанием времени
- 📊 Просмотр графика от администратора
- ✅ Отметка фактически отработанного времени
- 💰 Расчёт зарплаты по периодам
- 🏆 Просмотр своих приоритетов по аттракционам
- 🎯 Установка цели изучения нового аттракциона
- 🤖 Управление сменами через бота ВКонтакте

### Для администраторов
- 👥 Просмотр и управление сменами всех сотрудников
- ⚡ Автоматическая генерация графика на выбранный день
- ✏️ Ручное составление и редактирование графика
- 📈 Просмотр графика в режимах: день / неделя / месяц
- 🎡 Управление списком аттракционов
- 👤 Просмотр информации о сотрудниках
- 🏷️ Настройка приоритетов сотрудников по аттракционам

## 🚀 Быстрый старт

### Требования
- Node.js 18+
- npm или yarn
- Аккаунт Supabase

### Установка

```bash
# Клонировать репозиторий
git clone https://github.com/AlBarsStudio/park-staff.git
cd park-staff

# Установить зависимости
npm install
Настройка переменных окружения
Создать файл .env в корне проекта:

env

VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=your-anon-key
Значения берутся из панели управления Supabase:
Project Settings → API → Project URL и anon public key

Запуск
Bash

# Режим разработки
npm run dev

# Сборка для продакшна
npm run build

# Предпросмотр сборки
npm run preview
👥 Роли пользователей
Уровень	Роль	Доступ
3	Сотрудник	Личный кабинет, управление своими сменами
2	Администратор	Полная панель управления
Роль определяется автоматически при входе по записи в базе данных.
