## О проекте

**ParkStaff** — это система управления персоналом, разработанная для сотрудников парка аттракционов и администрации.

Платформа решает ключевые операционные задачи:

- сбор данных о доступности сотрудников на смены;
- упрощение составления рабочего графика;
- централизованное хранение информации о сотрудниках, аттракционах и назначениях;
- подготовка базы для расчёта заработной платы;
- интеграция сайта, базы данных и бота в беседе ВКонтакте.

Все данные хранятся в **Supabase**.  
Сайт предоставляет основной интерфейс для сотрудников и администраторов, а бот ВКонтакте позволяет записывать данные в базу без необходимости заходить на сайт.

---

## Назначение системы

ParkStaff создан для того, чтобы сократить ручную работу при ведении графика и повысить прозрачность взаимодействия между сотрудниками и администрацией.

### Что даёт система

- упрощает подачу смен сотрудниками;
- снижает нагрузку на администратора при составлении графика;
- позволяет быстрее видеть доступных сотрудников на нужную дату;
- помогает учитывать допуски и приоритеты по аттракционам;
- формирует основу для автоматизации расчёта зарплаты.

---

## Основные возможности

### Для сотрудников

- вход в личный кабинет;
- просмотр своих данных;
- добавление смен через календарь;
- удаление смен по установленным правилам;
- указание:
  - полной смены;
  - неполной смены с конкретным временем;
- просмотр графика, составленного администратором;
- отметка фактически отработанного времени;
- просмотр приоритетов по аттракционам;
- установка цели изучения нового аттракциона;
- просмотр расчёта заработной платы;
- работа через бота ВКонтакте.

### Для администраторов

- работа в административной панели;
- просмотр и управление сменами сотрудников;
- автоматическая генерация графика;
- ручное составление графика;
- просмотр графика в режимах день / неделя / месяц;
- управление списком сотрудников;
- управление аттракционами;
- настройка приоритетов сотрудников;
- ведение журнала действий.

## Роли пользователей

| Уровень доступа | Роль | Назначение |
|---|---|---|
| `2` | Администратор | Управление сменами, графиком, сотрудниками и аттракционами |
| `3` | Сотрудник | Работа только со своими сменами, графиком и личными данными |

> В системе также существует уровень Super Admin, однако он не включён в публичную пользовательскую документацию.

## Бизнес-правила

### Правила добавления смен

Сотрудник может добавить смену:

- на текущий день — только если текущее время **раньше 09:00**;
- на любую будущую дату.

Ограничения:

- нельзя добавить две смены на одну дату;
- при неполной смене обязательно указываются `start_time` и `end_time`;
- время окончания должно быть позже времени начала;
- комментарий ограничен длиной до `4096` символов.

### Правила удаления смен

Сотрудник не может удалить смену, если:

- дата смены уже наступила;
- смена относится к сегодняшнему дню;
- до начала смены осталось менее **22 часов**.

### Правила генерации графика

При составлении графика учитываются:

- доступность сотрудника на выбранную дату;
- наличие допуска к аттракциону;
- уровень приоритета сотрудника;
- минимально необходимое количество сотрудников на аттракцион;
- тип смены.

### Приоритеты распределения

- **полная смена** имеет приоритет над неполной;
- наличие аттракциона в приоритетах означает, что у сотрудника есть допуск;
- уровни приоритета:
  - `1` — высокий;
  - `2` — средний;
  - `3` — низкий.

### Правила отметки фактического времени

Сотрудник может отметить фактическое время работы только:

- после `22:00` в день смены;
- либо в любой момент после этой даты.

## Модель аутентификации

Система использует **Supabase Auth** для входа пользователей.

### Как определяется роль

1. Пользователь входит по email и паролю;
2. Supabase возвращает активную сессию;
3. Система ищет `auth_uid`:
   - сначала в таблице `admins`;
   - затем в таблице `employees`;
4. По найденной записи определяется роль и права доступа.

## Расчёт зарплаты

> Модуль расчёта заработной платы находится в активной разработке.

На текущем этапе реализована логика:

- расчёт по двум периодам:
  - `7–21`;
  - `22–6`;
- использование:
  - базовой часовой ставки сотрудника;
  - коэффициента аттракциона;
  - фактически отмеченного времени;
- оплата начинается с `11:00`, даже если сотрудник пришёл раньше;
- в расчёт попадают только смены, для которых сохранено фактическое время работы.
